## 4. 重点一：脚本为什么开始不稳定

### 4.1 一句话理解

不是 Selenium 突然变随机了，而是页面状态和脚本时机没对上。

### 4.2 今天最常见的 4 类问题

- 元素还没出现
- 元素出现了但不可见
- 元素存在但不可点击
- 页面重渲染导致旧元素失效

### 4.3 随堂检查

1. 页面上看得到按钮，就一定能点吗？
2. 同一段脚本有时成功有时报错，最常见是哪里出了问题？

### 4.4 答案

1. 不一定，它可能还不能点，或者被遮挡。
2. 最常见是页面状态和脚本执行时机没对上。

---

## 5. 重点二：显式等待

### 5.1 一句话理解

显式等待就是：等某个条件成立，再继续执行。

### 5.2 Day 3 最常见的条件

- `presence_of_element_located`
- `visibility_of_element_located`
- `element_to_be_clickable`
- `text_to_be_present_in_element`

### 5.3 最小示例

```python
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC
from selenium.webdriver.common.by import By

wait = WebDriverWait(driver, 10)
wait.until(EC.text_to_be_present_in_element((By.ID, "delayed-message"), "Hello Day 3"))
```

### 5.4 随堂检查

1. 显式等待和 `sleep()` 的核心区别是什么？
2. 等按钮能点，最常见用哪个条件？

### 5.5 答案

1. 显式等待在等“条件”，`sleep()` 只是死等固定时间。
2. `element_to_be_clickable`。

## 6. 重点三：断言

### 6.1 一句话理解

等待是为了让页面准备好，断言是为了确认结果对不对。

### 6.2 Day 3 常见断言

- 文本断言
- 属性断言
- 状态断言

### 6.3 例子

```python
assert "Hello Day 3" in driver.find_element(By.ID, "delayed-message").text
assert driver.find_element(By.ID, "delayed-action").is_enabled()
assert driver.find_element(By.ID, "action-result").get_attribute("data-clicked") == "true"
```

### 6.4 随堂检查

1. 没有断言时，你最多知道什么？
2. 等待和断言为什么不能混成一件事？

### 6.5 答案

1. 最多知道代码有没有报错。
2. 等待是在等条件成立，断言是在判断结果是否符合预期。

## 7. 重点四：高频异常先认名字

### 7.1 `NoSuchElementException`

更像：现在找不到元素。

### 7.2 `TimeoutException`

更像：等了，但没等到条件成立。

### 7.3 `StaleElementReferenceException`

更像：你手里的元素已经过期了。

### 7.4 `ElementClickInterceptedException`

更像：元素被挡住了。

### 7.5 覆盖层场景的处理思路

今天这页就有一个现成例子：

- 按钮：`covered-button`
- 覆盖层：`blocking-overlay`

如果你一打开页面就点击按钮，很可能会遇到点击拦截。

推荐顺序是：

1. 先理解它不是定位失败
2. 再等待覆盖层消失
3. 最后重新定位按钮并点击

### 7.5 随堂检查

1. `StaleElementReferenceException` 最像哪类问题？
2. `ElementClickInterceptedException` 最像在提示什么？

### 7.6 答案

1. 页面重渲染后，旧元素引用失效。
2. 目标元素被其他层挡住了。

### 8.5 参考答案

基础练习参考代码：

```python
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC

driver = webdriver.Edge()
driver.get("http://127.0.0.1:8000/pages/day3.html")

driver.find_element(By.ID, "start-loading").click()
wait = WebDriverWait(driver, 10)
wait.until(EC.text_to_be_present_in_element((By.ID, "delayed-message"), "Hello Day 3"))

message = driver.find_element(By.ID, "delayed-message").text
assert "Hello Day 3" in message
print(message)

driver.quit()
```

提升练习参考代码：

```python
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC

driver = webdriver.Edge()
driver.get("http://127.0.0.1:8000/pages/day3.html")

driver.find_element(By.ID, "enable-action").click()
wait = WebDriverWait(driver, 10)
button = wait.until(EC.element_to_be_clickable((By.ID, "delayed-action")))
button.click()

result = driver.find_element(By.ID, "action-result")
assert result.get_attribute("data-clicked") == "true"
print(result.text)

driver.quit()
```

覆盖层练习参考代码：

```python
from selenium import webdriver
from selenium.common.exceptions import ElementClickInterceptedException
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC

driver = webdriver.Edge()
driver.get("http://127.0.0.1:8000/pages/day3.html")

button = driver.find_element(By.ID, "covered-button")

try:
    button.click()
except ElementClickInterceptedException:
    wait = WebDriverWait(driver, 10)
    wait.until(EC.invisibility_of_element_located((By.ID, "blocking-overlay")))
    driver.find_element(By.ID, "covered-button").click()

result = driver.find_element(By.ID, "covered-result").text
assert "按钮点击成功" in result
print(result)

driver.quit()
```
