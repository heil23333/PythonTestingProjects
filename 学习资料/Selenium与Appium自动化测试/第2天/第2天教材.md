## 6. 先看懂 HTML 里的哪一块

很多初学者会混淆一件事：

Selenium 里的这些写法，到底是在 HTML 里看哪里。

先看一个最小例子：

```html
<input
  id="login-username"
  name="username"
  class="input-field primary"
  type="text"
  placeholder="输入用户名"
>

<a id="practice-link" href="#interaction-panel">跳转到交互区</a>

<button id="submit-login" type="button">提交登录表单</button>
```

你可以直接按下面这个表来对应：

| Selenium 写法 | 在 HTML 里看哪里 | 例子 |
| --- | --- | --- |
| `By.ID` | 看 `id="..."` | `login-username` |
| `By.NAME` | 看 `name="..."` | `username` |
| `By.CLASS_NAME` | 看 `class="..."` 里的单个类名 | `input-field` |
| `By.TAG_NAME` | 看标签名本身 | `input`、`a`、`button` |
| `By.LINK_TEXT` | 看链接标签中间显示出来的文字 | `跳转到交互区` |
| `By.CSS_SELECTOR` | 不是固定看一处，而是用 CSS 规则组合描述元素 | `#login-username` |
| `By.XPATH` | 不是固定看一处，而是用 XPath 规则组合描述元素 | `//input[@id="login-username"]` |

你可以先记这个口诀：

`id 看 id，name 看 name，class 看 class，tag 看标签名，link text 看显示文字。`

还要特别注意 2 个点：

1. `class name` 只能取单个类名  
   例如 `class="input-field primary"`，你只能用：

```python
driver.find_element(By.CLASS_NAME, "input-field")
```

不能直接写：

```python
driver.find_element(By.CLASS_NAME, "input-field primary")
```

2. `name` 不一定唯一  
   所以你看到 `name="language"` 时，不能默认页面里只有一个。

### 6.1 在 Day 2 页面里怎么看

拿本地 Day 2 页面举例：

```html
<input id="login-username" name="username" type="text" placeholder="输入用户名">
```

这里你可以这样对应：

- `By.ID` 对应 `login-username`
- `By.NAME` 对应 `username`
- `By.TAG_NAME` 对应 `input`
- `get_attribute("placeholder")` 对应 `输入用户名`

再看这个：

```html
<a id="practice-link" href="#interaction-panel">跳转到交互区</a>
```

这里你可以这样对应：

- `By.ID` 对应 `practice-link`
- `By.TAG_NAME` 对应 `a`
- `By.LINK_TEXT` 对应 `跳转到交互区`
- `get_attribute("href")` 对应链接地址

再看一个很适合自动化定位的例子：

```html
<p class="locator-note" data-testid="summary-note">这个段落适合练 css selector 和 xpath。</p>
<li class="skill-item" data-skill="id">稳定 id</li>
```

这里的：

- `data-testid`
- `data-skill`

都属于 `data-*` 自定义属性，也常叫“业务属性”或“测试属性”。

你要先记住两件事：

1. 只要你在 HTML 里看到 `data-` 开头的属性，它就可以成为自动化定位候选
2. `data-*` 不是单独的 `By.DATA`，通常配合 `CSS_SELECTOR` 或 `XPATH`

例如：

```python
driver.find_element(By.CSS_SELECTOR, '[data-testid="summary-note"]')
driver.find_element(By.CSS_SELECTOR, 'li[data-skill="id"]')
driver.find_element(By.XPATH, '//*[@data-testid="summary-note"]')
driver.find_element(By.XPATH, '//li[@data-skill="id"]')
```

为什么它很有用：

- 通常比样式类名更稳定
- 更能表达业务含义
- 很适合专门给自动化留定位锚点

### 6.2 随堂检查

1. `data-*` 在 HTML 里一般长什么样？
2. 为什么 `data-*` 常常比样式类名更适合自动化定位？
3. `data-testid="summary-note"` 这种属性通常配合什么方式来定位？

### 6.3 答案

1. 一般就是 `data-` 开头的属性，例如 `data-testid="summary-note"`、`data-skill="id"`。
2. 因为它通常更稳定，也更像团队专门留给业务和测试识别的锚点。
3. 通常配合 `By.CSS_SELECTOR` 或 `By.XPATH`。

## 7. Day 2 要掌握的定位方式

### 7.1 `By.ID`

这是最优先考虑的定位方式之一。

示例：

```python
driver.find_element(By.ID, "login-username")
```

为什么优先：

- 通常更短
- 语义更清晰
- 一般更稳定

什么时候适合用：

- 页面上有明确且稳定的 `id`

什么时候不适合盲信：

- 页面里 `id` 是随机生成的
- 每次刷新页面 `id` 都变

### 7.2 `By.NAME`

`name` 也很常见，但它不一定唯一。

示例：

```python
driver.find_element(By.NAME, "username")
```

你要先记住一件事：

`name` 可以重复，所以它不是天然唯一定位。

例如单选框就经常共用同一个 `name`。

### 7.3 `By.CLASS_NAME`

示例：

```python
driver.find_element(By.CLASS_NAME, "skill-item")
```

它适合：

- 某一类相似元素
- 批量查找

不适合：

- 类名经常变
- 你误把多个类名一起传进去

注意：

`class name` 只能传单个类名，不能把带空格的整串类名直接扔进去。

### 7.4 `By.TAG_NAME`

示例：

```python
driver.find_element(By.TAG_NAME, "button")
```

它适合：

- 页面里该标签很少
- 你只是临时验证某个通用按钮

它不适合长期作为核心定位方式，因为一个页面上通常会有多个同标签元素。

### 7.5 `By.LINK_TEXT`

示例：

```python
driver.find_element(By.LINK_TEXT, "跳转到交互区")
```

适合：

- 链接文字稳定
- 页面文案不会频繁改

不适合：

- 产品文案经常调整
- 文本存在多语言切换

### 7.6 `By.CSS_SELECTOR`

示例：

```python
driver.find_element(By.CSS_SELECTOR, "#login-username")
driver.find_element(By.CSS_SELECTOR, '[data-testid="summary-note"]')
driver.find_element(By.CSS_SELECTOR, "#skills-list .skill-item")
```

为什么很重要：

- 表达力强
- 写法通常比 `xpath` 更简洁
- 实战里非常常用

适合：

- 用 `id`、属性、层级做简洁组合
- 需要稳定但不想写太长

如果页面上有 `data-*`，这是 `css selector` 很常见、也很推荐的写法：

```python
driver.find_element(By.CSS_SELECTOR, '[data-testid="summary-note"]')
driver.find_element(By.CSS_SELECTOR, 'li[data-skill="id"]')
```

### 7.7 `By.XPATH`

示例：

```python
driver.find_element(By.XPATH, '//p[@data-testid="summary-note"]')
driver.find_element(By.XPATH, '//label[contains(., "同意练习站使用说明")]')
```

`xpath` 不是不能用，但不要默认最优先。

它适合：

- 需要按文本定位
- 需要处理父子、兄弟关系
- 没有更好的稳定属性可用

如果你想用 `xpath` 定位 `data-*`，常见写法是：

```python
driver.find_element(By.XPATH, '//*[@data-testid="summary-note"]')
driver.find_element(By.XPATH, '//li[@data-skill="id"]')
```

不推荐一上来就写这种长路径：

```python
/html/body/main/section[2]/div[1]/div[2]/ul/li[2]
```

这种定位过度依赖页面结构，一旦多加一层容器就容易失效。

### 7.8 随堂检查

1. 哪两种定位方式通常最适合作为优先候选？
2. 为什么 `class name` 定位要谨慎？
3. 如果要按链接文案定位，应该优先想到哪种方式？
4. 如果要定位 `data-testid="summary-note"`，你至少能写出哪两种常见写法？

### 7.9 答案

1. 通常优先考虑 `id` 和稳定的 `name` 或 `data-*` 属性。
2. 因为类名常常为样式服务，变化频率可能更高，而且多个元素可能共享同类名。
3. `By.LINK_TEXT`。
4. 例如 `By.CSS_SELECTOR, '[data-testid="summary-note"]'`，或者 `By.XPATH, '//*[@data-testid="summary-note"]'`。

## 8. 定位方式优先顺序

Day 2 先把这个顺序记住：

`id > name > data-* > css > xpath`

你也可以记口诀：

`先 id，再 name；再看稳定属性，css 常上；关系复杂，最后 xpath。`

这里的“稳定属性”包括：

- `data-testid`
- `data-test`
- `data-qa`
- 其他团队约定好的 `data-*`

这些属性在 HTML 里看起来通常长这样：

```html
data-testid="summary-note"
data-skill="id"
```

判断一个定位是不是好定位，可以看这 3 点：

1. 会不会因为页面层级变化就挂
2. 会不会因为样式类名变化就挂
3. 这个定位是不是一眼能看出业务含义

## 9. Day 2 常用页面交互

今天只讲定位之后紧跟着会用到的交互，不引入更复杂的等待和异常处理。

### 9.1 `send_keys()`

作用：

- 给输入框输入内容

示例：

```python
driver.find_element(By.ID, "login-username").send_keys("selenium_student")
```

### 9.2 `clear()`

作用：

- 清空输入框现有内容

示例：

```python
driver.find_element(By.ID, "login-username").clear()
```

### 9.3 `click()`

作用：

- 点击按钮、复选框、单选框、链接

示例：

```python
driver.find_element(By.ID, "fill-demo").click()
```

### 9.4 `.text`

作用：

- 获取元素中显示给用户看的文本

示例：

```python
status = driver.find_element(By.ID, "login-status").text
```

### 9.5 `get_attribute()`

作用：

- 获取元素属性值

示例：

```python
value = driver.find_element(By.ID, "login-username").get_attribute("value")
filled = driver.find_element(By.ID, "login-username").get_attribute("data-filled")
```

如果你拿输入框内容，通常更应该用：

```python
get_attribute("value")
```

### 9.6 常见属性怎么取

Day 2 很容易混淆的一点是：

- 什么该用 `.text`
- 什么该用 `get_attribute()`
- 什么更适合用状态方法

你可以先按下面这个表来记：

| 想拿什么 | 推荐写法 | 说明 |
| --- | --- | --- |
| 元素显示文本 | `.text` | 取页面上显示给用户看的文字 |
| `id` | `get_attribute("id")` | 读取元素 `id` 属性 |
| `name` | `get_attribute("name")` | 读取元素 `name` 属性 |
| `class` | `get_attribute("class")` | 读取完整类名字符串 |
| `type` | `get_attribute("type")` | 常用于输入框、按钮 |
| `placeholder` | `get_attribute("placeholder")` | 常用于输入框提示文案 |
| 输入框当前值 | `get_attribute("value")` | 输入框最常用 |
| 链接地址 | `get_attribute("href")` | 常用于 `<a>` 标签 |
| 自定义业务属性 | `get_attribute("data-testid")` | `data-*` 同理 |
| 勾选状态 | `is_selected()` | 复选框、单选框更推荐 |
| 是否显示 | `is_displayed()` | 判断可见性 |
| 是否可用 | `is_enabled()` | 判断能不能操作 |

在 Day 2 页面里可以直接练这些：

```python
username = driver.find_element(By.ID, "login-username")
print(username.get_attribute("id"))
print(username.get_attribute("name"))
print(username.get_attribute("type"))
print(username.get_attribute("placeholder"))
print(username.get_attribute("value"))

link = driver.find_element(By.ID, "practice-link")
print(link.get_attribute("href"))

note = driver.find_element(By.CSS_SELECTOR, '[data-testid="summary-note"]')
print(note.get_attribute("data-testid"))

checkbox = driver.find_element(By.ID, "agree-policy")
print(checkbox.is_selected())
```

你要特别记住 3 组最容易混淆的点：

1. 输入框当前内容
   - 常用 `get_attribute("value")`
   - 不要默认用 `.text`
2. 链接地址
   - 用 `get_attribute("href")`
3. 勾选状态
   - 复选框和单选框优先用 `is_selected()`

### 9.7 `is_displayed()` 和 `is_enabled()`

作用：

- 判断元素是否显示
- 判断元素是否可用

示例：

```python
driver.find_element(By.ID, "submit-login").is_displayed()
driver.find_element(By.ID, "submit-login").is_enabled()
```

### 9.8 `select` 下拉框怎么操作

如果页面元素是原生 `<select>`，最推荐的方式不是自己乱点，而是用 Selenium 自带的 `Select`。

先导入：

```python
from selenium.webdriver.support.select import Select
```

再定位下拉框：

```python
select_el = driver.find_element(By.ID, "target-level")
select = Select(select_el)
```

最常用的 3 种选法：

1. 按显示文字选

```python
select.select_by_visible_text("Day 3")
```

2. 按 `value` 选

```python
select.select_by_value("day3")
```

3. 按下标选

```python
select.select_by_index(1)
```

如果你想看当前选中了什么：

```python
print(select.first_selected_option.text)
print(select.first_selected_option.get_attribute("value"))
```

拿 Day 2 页面举例：

```html
<select id="target-level" name="target_level">
  <option value="day2">Day 2</option>
  <option value="day3">Day 3</option>
  <option value="day4">Day 4</option>
</select>
```

你可以这样写：

```python
from selenium.webdriver.support.select import Select

select = Select(driver.find_element(By.ID, "target-level"))
select.select_by_visible_text("Day 3")
print(select.first_selected_option.text)
```

实战里先记一句：

`原生 select 用 Select，不要默认靠 click 硬点。`

## 13. 自测题

### 13.1 概念题

1. 为什么 `id` 往往比 `xpath` 更适合作为首选定位？
2. 为什么 `name` 不能默认唯一？
3. 输入框里的当前值为什么常用 `get_attribute("value")`，而不是 `.text`？
4. 如果你想拿链接跳转地址，应该优先用什么写法？
5. 如果你想判断复选框是否勾选，应该优先用什么写法？
6. `By.TAG_NAME` 在 HTML 里对应看什么？
7. `By.LINK_TEXT` 在 HTML 里对应看什么？
8. 如果页面是原生 `<select>`，推荐用什么方式操作？

### 13.2 代码理解题

看下面代码，回答问题：

```python
from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.support.select import Select

driver = webdriver.Edge()
driver.get("http://127.0.0.1:8000/pages/day2.html")

driver.find_element(By.ID, "fill-demo").click()
value = driver.find_element(By.ID, "login-username").get_attribute("value")
text = driver.find_element(By.ID, "attribute-status").text
h = driver.find_element(By.ID, "practice-link").get_attribute("href")
select = Select(driver.find_element(By.ID, "target-level"))
select.select_by_value("day4")
print(value)
print(text)
print(h)
print(select.first_selected_option.text)

driver.quit()
```

请回答：

1. `fill-demo` 按钮做了什么？
2. `value` 和 `text` 分别拿到的是什么？
3. 如果想清空用户名，应该调用哪个方法？
4. `h` 拿到的是什么？
5. `select.first_selected_option.text` 拿到的是什么？

### 13.3 标准答案

1. 因为 `id` 通常更短、更稳定、更不依赖页面结构；而很多 `xpath` 容易过长、过脆。
2. 因为多个元素可以共用同一个 `name`，例如单选框。
3. 因为输入框内容通常保存在 `value` 属性里，不是作为可见文本节点直接存在。
4. 如果想拿链接地址，优先用 `get_attribute("href")`。
5. 如果想判断复选框或单选框是否勾选，优先用 `is_selected()`。
6. `By.TAG_NAME` 对应看 HTML 里的标签名本身，例如 `input`、`button`、`a`。
7. `By.LINK_TEXT` 对应看链接标签中间显示出来的文字。
8. 如果页面是原生 `<select>`，推荐用 Selenium 自带的 `Select`。
9. `fill-demo` 会给用户名和密码输入框写入演示值，并给用户名输入框追加 `data-filled=true`。
10. `value` 是用户名输入框的当前值，`text` 是属性状态区域显示给用户看的文字。
11. 应该调用 `clear()`。
12. `h` 拿到的是链接元素的跳转地址，也就是 `href` 属性值。
13. 它拿到的是当前下拉框选中项显示出来的文字，这里会是 `Day 4`。

## 14. 补充：常见特殊情况速查

这一节的目标不是让你今天全部掌握，而是先建立一个判断：

“看到这种元素时，不要按普通按钮或普通文本去硬点。”

### 14.1 今天就该掌握的 4 类

| 场景 | 推荐方式 | 为什么 |
| --- | --- | --- |
| 输入框 | `send_keys()` / `clear()` / `get_attribute("value")` | 输入框当前值通常不靠 `.text` 拿 |
| 复选框、单选框 | `click()` + `is_selected()` | 要判断是否真的选中 |
| 原生 `<select>` | `Select(...)` | 这是 Selenium 为下拉框准备的专用方式 |
| 普通可见文本 | `.text` | 读取页面上显示给用户看的文字 |

### 14.2 今天先认识，后面再深入的 6 类

| 场景 | 推荐方式 | 详细学习时间 |
| --- | --- | --- |
| 文件上传 `input[type="file"]` | `send_keys(绝对路径)` | Day 4 |
| `iframe` | `switch_to.frame()` | Day 4 |
| 浏览器弹窗 `alert/confirm/prompt` | `switch_to.alert` | Day 4 |
| 新窗口 / 新标签页 | `switch_to.window()` | Day 4 |
| 自定义下拉框 | 当普通元素处理，先点展开再点选项 | Day 4 |
| 隐藏 / 禁用元素 | 结合 `is_displayed()`、`is_enabled()`、等待机制 | Day 3 |

### 14.3 最容易误判的一类：自定义下拉框

很多页面看起来像下拉框，但 HTML 里根本不是原生 `<select>`。

如果它真实结构是：

- `div`
- `ul`
- `li`
- `span`

这种就不要用 `Select`。

正确思路通常是：

1. 先点击展开区域
2. 再定位列表项
3. 再点击具体选项

也就是说：

`只有原生 <select> 才优先用 Select。`

### 14.4 最容易误判的另一类：文件上传

很多人看到上传按钮会本能地想：

- 先 click
- 再处理系统文件选择框

但 Web 自动化里更常见、更稳的方式是：

```python
driver.find_element(By.ID, "upload").send_keys("/abs/path/demo.txt")
```

也就是说：

`文件上传通常不是点系统窗口，而是把绝对路径直接送给文件输入框。`

### 14.5 一句总原则

先看真实 HTML，再决定操作方式。

你可以记成：

`先判断元素类型，再决定是 click、Select、switch_to 还是 send_keys。`
