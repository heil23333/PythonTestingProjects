## 4. 重点一：先会找，再会点

### 4.1 一句话理解

Selenium 的所有页面操作，都建立在“先找到元素”这个前提上。

### 4.2 今天先记住的优先顺序

`id > name > data-* > css > xpath`

### 4.3 口诀

`先 id，再 name；再看稳定属性，css 常上；关系复杂，最后 xpath。`

### 4.4 随堂检查

1. 为什么定位要先学，不能直接先学点击？
2. 为什么 `xpath` 不应该默认最优先？

### 4.5 答案

1. 因为所有点击、输入、读取文本动作都要先找到元素，定位错了后面的动作就没有意义。
2. 因为很多 `xpath` 过长、过度依赖层级结构，页面一改就容易失效。

## 5. 重点二：先看 HTML 里哪一块

今天你一定要先建立这个对应关系。

看下面这段 HTML：

```html
<input id="login-username" name="username" class="input-field primary" type="text">
<a id="practice-link" href="#interaction-panel">跳转到交互区</a>
<button id="submit-login" type="button">提交登录表单</button>
```

你可以这样看：

- `By.ID`：看 `id="..."`
- `By.NAME`：看 `name="..."`
- `By.CLASS_NAME`：看 `class="..."` 里的单个类名
- `By.TAG_NAME`：看标签名本身，比如 `input`、`a`、`button`
- `By.LINK_TEXT`：看 a 标签中间显示出来的文字

再补一个很实用的：

- `data-*`：看所有 `data-` 开头的属性，比如 `data-testid`、`data-skill`

但要注意：

`data-*` 不是单独的 `By.DATA`。

它通常配合：

- `By.CSS_SELECTOR`
- `By.XPATH`

来使用。

一句话记忆：

`id 看 id，name 看 name，class 看 class，tag 看标签名，link text 看显示文字。`

### 5.1 随堂检查

1. `By.TAG_NAME` 在 HTML 里看哪里？
2. `By.LINK_TEXT` 在 HTML 里看哪里？
3. `By.CLASS_NAME` 为什么不能直接把多个类名整串传进去？
4. `data-testid="summary-note"` 这种属性通常配合什么方式来定位？

### 5.2 答案

1. 看标签名本身，比如 `input`、`button`、`a`。
2. 看链接标签里显示给用户看的文字。
3. 因为 `class name` 定位一次只能接收单个类名，不支持整串带空格的类名字符串。
4. 通常配合 `By.CSS_SELECTOR` 或 `By.XPATH`。

## 6. 重点三：7 种常见定位方式

### 6.1 `By.ID`

示例：

```python
driver.find_element(By.ID, "login-username")
```

特点：

- 短
- 清晰
- 通常稳定

### 6.2 `By.NAME`

示例：

```python
driver.find_element(By.NAME, "username")
```

重点提醒：

`name` 不一定唯一。

### 6.3 `By.CLASS_NAME`

示例：

```python
driver.find_element(By.CLASS_NAME, "skill-item")
```

重点提醒：

只能传一个类名，不能把带空格的整串类名直接传进去。

### 6.4 `By.TAG_NAME`

示例：

```python
driver.find_element(By.TAG_NAME, "button")
```

重点提醒：

通用性强，但唯一性通常较差。

### 6.5 `By.LINK_TEXT`

示例：

```python
driver.find_element(By.LINK_TEXT, "跳转到交互区")
```

重点提醒：

文字变了，定位通常也会跟着失效。

### 6.6 `By.CSS_SELECTOR`

示例：

```python
driver.find_element(By.CSS_SELECTOR, "#skills-list .skill-item")
```

重点提醒：

它很常用，通常比 `xpath` 更简洁。

如果页面上有 `data-*`，这是非常常见的写法：

```python
driver.find_element(By.CSS_SELECTOR, '[data-testid="summary-note"]')
driver.find_element(By.CSS_SELECTOR, 'li[data-skill="id"]')
```

### 6.7 `By.XPATH`

示例：

```python
driver.find_element(By.XPATH, '//p[@data-testid="summary-note"]')
```

重点提醒：

强大，但不要滥用长路径。

如果你想用 `xpath` 写 `data-*`，常见写法是：

```python
driver.find_element(By.XPATH, '//*[@data-testid="summary-note"]')
driver.find_element(By.XPATH, '//li[@data-skill="id"]')
```

### 6.8 随堂检查

1. 哪两种定位方式通常最适合作为优先候选？
2. 为什么 `class name` 定位要谨慎？
3. 如果要按链接文案定位，应该优先想到哪种方式？
4. 如果要定位 `data-testid="summary-note"`，你至少能写出哪两种常见写法？

### 6.9 答案

1. 通常优先考虑 `id` 和稳定的 `name` 或 `data-*` 属性。
2. 因为类名常常为样式服务，变化频率可能更高，而且多个元素可能共享同类名。
3. `By.LINK_TEXT`。
4. 例如 `By.CSS_SELECTOR, '[data-testid="summary-note"]'`，或者 `By.XPATH, '//*[@data-testid="summary-note"]'`。

## 7. 重点四：页面交互

### 7.1 `send_keys()`

对应用户输入。

### 7.2 `clear()`

对应清空输入框。

### 7.3 `click()`

对应点击按钮、勾选、切换单选框、点击链接。

### 7.4 `.text`

读取页面里显示出来的文字。

### 7.5 `get_attribute()`

读取属性值，比如：

- `value`
- `data-filled`
- `data-role`

### 7.6 常见属性获取速记

今天先记这张小表：

| 想拿什么 | 推荐写法 |
| --- | --- |
| 可见文字 | `.text` |
| 输入框当前值 | `get_attribute("value")` |
| 占位提示 | `get_attribute("placeholder")` |
| 链接地址 | `get_attribute("href")` |
| 自定义业务属性 | `get_attribute("data-testid")` |
| 勾选状态 | `is_selected()` |
| 元素是否显示 | `is_displayed()` |
| 元素是否可用 | `is_enabled()` |

### 7.7 `is_displayed()` 和 `is_enabled()`

判断元素是否显示、是否可用。

### 7.8 `select` 下拉框怎么操作

如果页面是原生 `<select>`，优先用 Selenium 的 `Select`。

先导入：

```python
from selenium.webdriver.support.select import Select
```

再这样写：

```python
select = Select(driver.find_element(By.ID, "target-level"))
select.select_by_visible_text("Day 3")
print(select.first_selected_option.text)
```

你可以记 3 种常见选法：

- `select_by_visible_text()`
- `select_by_value()`
- `select_by_index()`

### 7.9 随堂检查

1. 输入框当前内容为什么更适合用 `get_attribute("value")`？
2. `.text` 更像是在拿什么？
3. `clear()` 的使用场景是什么？
4. 想拿链接地址，应该优先用什么？
5. 想判断复选框是否勾选，应该优先用什么？
6. 原生 `<select>` 推荐怎么操作？

### 7.10 答案

1. 因为输入框当前值通常存在 `value` 属性里，而不是直接作为元素内部文本存在。
2. 更像是在拿“页面上显示给用户看的文字”。
3. 当你要清空已有输入，重新输入新值时就要用 `clear()`。
4. 优先用 `get_attribute("href")`。
5. 优先用 `is_selected()`。
6. 优先用 Selenium 的 `Select`。

## 11. 特殊情况速记

今天先记结论，不要求全部会写。

### 11.1 先分清元素类型

- 输入框：`send_keys()`、`clear()`、`get_attribute("value")`
- 复选框 / 单选框：`click()`、`is_selected()`
- 原生 `<select>`：`Select`
- 普通文本：`.text`

### 11.2 后面会深入的特殊场景

- 文件上传：通常是给文件输入框 `send_keys(绝对路径)`
- `iframe`：先 `switch_to.frame()`
- 浏览器弹窗：先 `switch_to.alert`
- 新窗口 / 新标签页：先切 `window`
- 自定义下拉框：按普通元素点击，不用 `Select`
- 隐藏或禁用元素：结合等待和状态判断

### 11.3 一句总原则

`先看真实 HTML，再决定操作方式。`

如果它不是原生 `<select>`，就不要默认用 `Select`。
