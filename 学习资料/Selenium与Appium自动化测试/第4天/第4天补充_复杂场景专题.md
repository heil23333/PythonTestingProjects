## 3. Scenario 1：iframe

### 3.1 先建立整体认知

`iframe` 可以先理解成：

`页面里的另一个页面。`

它的核心难点不是定位语法，而是：

`你当前到底在哪个 DOM 上下文里。`

你要先记住 4 个 API：

- `driver.switch_to.frame(...)`
- `driver.switch_to.parent_frame()`
- `driver.switch_to.default_content()`
- `EC.frame_to_be_available_and_switch_to_it(...)`

---

### 3.2 `driver.switch_to.frame(frame_reference)`

#### 它是什么

这是一个**方法**，作用是：

`切进某个 iframe。`

#### 参数能传什么

常见有 3 类：

1. `iframe` 的索引

```python
driver.switch_to.frame(0)
```

这里的 `0` 不是随便写的，它表示：

`当前页面上下文里第 1 个 iframe`

索引从 `0` 开始算，也就是：

- `0`：第 1 个 iframe
- `1`：第 2 个 iframe
- `2`：第 3 个 iframe

2. `iframe` 的 `name` 或 `id`

```python
driver.switch_to.frame("day4-practice-frame")
```

3. `iframe` 对应的 `WebElement`

```python
frame_el = driver.find_element(By.ID, "day4-practice-frame")
driver.switch_to.frame(frame_el)
```

#### 最推荐哪种

学习阶段更推荐：

- `id/name`
- 或 `WebElement`

因为它比纯索引更清楚。

索引能用，但页面上 iframe 一多就不稳定。

#### 这个索引到底怎么获取

最本质的答案是：

`它不是一个单独写在 HTML 里的属性，而是 iframe 在当前 DOM 上下文中的顺序位置。`

也就是说，你不能像 `id` 一样直接去看：

```html
<iframe index="0">
```

页面里通常没有这种东西。

你要理解成：

- Selenium 看到当前上下文里有多个 iframe
- 它按出现顺序给它们排位置
- 第一个就是 `0`
- 第二个就是 `1`

#### 怎么自己确认索引

方法 1：看开发者工具里的顺序

如果当前页面 HTML 是：

```html
<iframe id="frame-a"></iframe>
<iframe id="frame-b"></iframe>
```

那通常就是：

- `frame-a` 对应索引 `0`
- `frame-b` 对应索引 `1`

方法 2：先把当前页面里的 iframe 全找出来

```python
frames = driver.find_elements(By.TAG_NAME, "iframe")
print(len(frames))
print(frames[0].get_attribute("id"))
```

如果你想先看“第 0 个到底是谁”，这个方法最直接。

#### 返回什么

这个方法一般也不拿来接返回值。

你把它理解成：

`切换当前上下文`

就够了。

#### 什么时候用

只要目标元素在 `iframe` 里面，你就必须先切。

否则你在主页面写：

```python
driver.find_element(By.ID, "frame-username")
```

通常直接报找不到元素。

---

### 3.3 `driver.switch_to.default_content()`

#### 它是什么

这是一个**方法**，作用是：

`切回最外层主页面。`

#### 什么时候用

当你已经在 iframe 里操作完，准备回主页面继续找元素时，就要切回来。

例如：

```python
driver.switch_to.default_content()
driver.find_element(By.ID, "advanced-host-note")
```

#### 为什么必须切回来

因为如果你还停留在 iframe 上下文里，后面再找主页面元素仍然会失败。

---

### 3.4 `driver.switch_to.parent_frame()`

#### 它是什么

这是一个**方法**，作用是：

`从当前 iframe 回到上一级 frame。`

#### 它和 `default_content()` 的区别

- `parent_frame()`：回上一层
- `default_content()`：直接回最外层

如果你只有一层 iframe，这两个效果可能看起来差不多。

但如果以后遇到嵌套 iframe：

- `parent_frame()` 更细
- `default_content()` 更直接

#### Day 4 先怎么记

Day 4 先优先掌握：

- 进 iframe：`switch_to.frame(...)`
- 回主页面：`switch_to.default_content()`

`parent_frame()` 先知道名字和区别就够。

---

### 3.5 `EC.frame_to_be_available_and_switch_to_it(locator)`

#### 它是什么

这是一个 expected condition。

它最重要的价值是：

`等待 iframe 可用，并直接切进去。`

#### 写法

```python
wait.until(EC.frame_to_be_available_and_switch_to_it((By.ID, "day4-practice-frame")))
```

#### 参数是什么

通常传：

- 定位元组 `(By.XXX, "value")`
- 或某个 frame 引用

学习阶段最推荐你传定位元组。

#### 返回什么

你现在不用重点记它返回什么，更重要的是记住：

`它成功后，你已经在 iframe 上下文里了。`

#### 为什么这个方法很适合初学者

因为它把两步合成一步了：

1. 等 frame 可用
2. 直接切进去

比你自己先找 iframe 再切，通常更稳。

---

### 3.6 iframe 完整示例

```python
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC


driver.get("http://127.0.0.1:8000/pages/day4_advanced.html")

wait = WebDriverWait(driver, 10)
wait.until(EC.frame_to_be_available_and_switch_to_it((By.ID, "day4-practice-frame")))

driver.find_element(By.ID, "frame-username").send_keys("frame_user")
driver.find_element(By.ID, "frame-submit").click()

result = driver.find_element(By.ID, "frame-result")
assert "iframe 提交成功" in result.text
assert result.get_attribute("data-user") == "frame_user"

driver.switch_to.default_content()
assert "主页面上下文" in driver.find_element(By.ID, "advanced-host-note").text
```

#### 这段代码要读出什么

1. 不是先定位输入框，而是先切 frame
2. 切进去后，iframe 内部元素就能正常找了
3. 操作完要切回主页面

---

### 3.7 最容易报什么错

- `NoSuchElementException`
  - 原因：没切 frame 就去找 iframe 里的元素
- `NoSuchFrameException`
  - 原因：frame 定位写错

这也是为什么 Day 4 推荐你优先用：

```python
EC.frame_to_be_available_and_switch_to_it(...)
```

## 4. Scenario 2：浏览器原生弹窗

### 4.1 先建立整体认知

浏览器原生弹窗包括：

- `alert`
- `confirm`
- `prompt`

这类弹窗不是普通 HTML 元素。

所以它们不能靠：

```python
driver.find_element(...)
```

去找。

你要走的是 Selenium 给弹窗专门准备的入口：

- `driver.switch_to.alert`

---

### 4.2 `driver.switch_to.alert`

#### 它是什么

这是一个**属性**，不是方法。

写法：

```python
alert = driver.switch_to.alert
```

不是：

```python
alert = driver.switch_to.alert()
```

#### 它返回什么

返回一个 Alert 对象。

你后面常用的操作都在这个对象上：

- `alert.text`
- `alert.accept()`
- `alert.dismiss()`
- `alert.send_keys()`

#### 什么时候用

当原生弹窗已经出现后，再去拿它。

例如：

```python
driver.find_element(By.ID, "trigger-alert").click()
alert = driver.switch_to.alert
```

#### 最容易报什么错

- `NoAlertPresentException`

常见原因：

- 弹窗还没出来
- 其实当前页面没有原生弹窗

---

### 4.3 `alert.text`

#### 它是什么

这是一个**属性**，不是方法。

写法：

```python
message = alert.text
```

#### 它返回什么

返回弹窗里显示的文本。

#### 什么时候用

最常见就是先断言文案，再决定是 `accept()` 还是 `dismiss()`。

例如：

```python
assert "Day 4 alert" in alert.text
```

---

### 4.4 `alert.accept()`

#### 它是什么

这是一个**方法**。

作用是：

`点击弹窗里的“确定 / 接受”`

#### 什么时候用

- `alert`：通常就是确认
- `confirm`：表示点“确定”
- `prompt`：输入后确认

写法：

```python
alert.accept()
```

#### 返回什么

通常不拿来接返回值。

把它理解成“执行确认动作”即可。

---

### 4.5 `alert.dismiss()`

#### 它是什么

这是一个**方法**。

作用是：

`点击弹窗里的“取消 / 关闭”`

#### 什么时候用

最常见场景是：

- `confirm`
- `prompt`

例如：

```python
confirm.dismiss()
```

#### 注意

普通 `alert` 只有确认意义时，你通常不会用 `dismiss()`。

---

### 4.6 `alert.send_keys(text)`

#### 它是什么

这是一个**方法**。

作用是：

`向 prompt 弹窗输入文本`

#### 参数是什么

传一个字符串即可：

```python
prompt.send_keys("day4")
```

#### 什么时候用

只在 `prompt` 场景里最常见。

#### 注意

你一般会这样连着用：

```python
prompt.send_keys("day4")
prompt.accept()
```

因为只输入不确认，结果通常不会真正提交。

---

### 4.7 三类弹窗完整示例

#### alert

```python
from selenium.webdriver.common.by import By


driver.get("http://127.0.0.1:8000/pages/day4_advanced.html")
driver.find_element(By.ID, "trigger-alert").click()

alert = driver.switch_to.alert
assert "Day 4 alert" in alert.text
alert.accept()

assert "已处理完成" in driver.find_element(By.ID, "alert-result").text
```

#### confirm

```python
driver.find_element(By.ID, "trigger-confirm").click()

confirm = driver.switch_to.alert
assert "继续当前练习" in confirm.text
confirm.dismiss()

assert "你点击了取消" in driver.find_element(By.ID, "confirm-result").text
```

如果你想点确定，就改成：

```python
confirm.accept()
```

#### prompt

```python
driver.find_element(By.ID, "trigger-prompt").click()

prompt = driver.switch_to.alert
assert "请输入课程暗号" in prompt.text
prompt.send_keys("day4")
prompt.accept()

result = driver.find_element(By.ID, "prompt-result")
assert "day4" in result.text
assert result.get_attribute("data-value") == "day4"
```

### 4.11 建议练习步骤

这块建议按 `alert -> confirm -> prompt` 的顺序练。

原因是：

- `alert` 最简单，只要获取、读文本、确认
- `confirm` 会多一个“确定 / 取消”的分支

1. 先练 `trigger-alert`
2. 再练 `trigger-confirm`
3. 最后练 `trigger-prompt`
4. 每种都练“弹窗处理前”和“处理后页面结果断言”

## 5. Scenario 3：文件上传

### 5.1 先建立整体认知

这一块最重要的一句是：

`文件上传通常不是点系统文件选择框，而是直接给文件输入框 send_keys(绝对路径)。`

因为真正负责接收文件的，不是你看见的“上传按钮样式”，而是底层的：

```html
<input type="file">
```

Selenium 可以直接操作这个真实输入框。

---

### 5.2 `element.send_keys(path)` 在文件上传里的用法

你前面已经学过：

```python
input_el.send_keys("abc")
```

它可以给文本框输入内容。

在文件上传里，它的用法变成：

```python
file_input.send_keys("/绝对路径/xxx.txt")
```

也就是：

`直接把文件路径送给文件输入框。`

#### 参数有什么要求

最稳的是传：

- 绝对路径

例如：

```python
from pathlib import Path

file_path = Path("/Users/Apple/Documents/Codes/PythonTestingProjects/练习项目/Selenium与Appium本地练习站/static/day4_upload_template.txt")
file_input.send_keys(str(file_path))
```

为什么强调绝对路径：

- 相对路径在不同工作目录下容易错
- 绝对路径更稳定，排查也更方便

---

### 5.3 上传练习页里的相关元素

在复杂场景页里：

- 文件输入框：`upload-input`
- 结果区：`upload-result`

结果区会把这些状态写出来：

- 文本内容
- `data-file-name`

所以你既可以断言文本，也可以断言属性。

---

### 5.4 文件上传完整示例

### 5.5 为什么不建议练“点开系统文件窗口”

因为系统文件选择窗口通常不属于浏览器页面 DOM。

Selenium 更稳定的思路是：

`绕过系统弹窗，直接操作 input[type="file"]。`

这也是为什么面试和实战里一说文件上传，第一反应通常就是：

```python
send_keys(绝对路径)
```

## 6. Scenario 4：文件下载与多标签页 / 多窗口切换

这一块页面上放在同一个场景里，所以教材也放在一起讲。

你在页面上能同时练到：

- 下载链接的属性断言
- 点击链接后新标签页切换

---

### 6.1 文件下载先建立什么认知

在 Day 4 你先不要把下载想得太轻。

“点击下载链接”很简单，但“确认文件真的下载到本地”通常没那么简单。

因为它往往还要处理：

- 浏览器默认下载目录
- 是否弹出保存提示
- Headless 浏览器的下载设置
- 测试前后文件清理

所以 Day 4 更合理的目标是：

1. 能定位下载链接
2. 能断言 `href`
3. 能知道如果以后要做“落盘验证”，要加浏览器配置

---

### 6.2 `get_attribute("href")` 和 `get_attribute("download")`

下载链接最常见先看这两个属性：

#### `get_attribute("href")`

作用：

拿到下载链接真实指向的地址。

例如：

```python
download_link.get_attribute("href")
```

#### `get_attribute("download")`

作用：

查看这个链接有没有 `download` 属性。

例如：

```python
download_link.get_attribute("download")
```

注意：

有些页面里这个属性是空字符串，也依然表示它存在。

所以不要机械地只认某个固定返回值，最好结合当前页面实际 HTML 来看。

---

### 6.3 下载入口示例

```python
from selenium.webdriver.common.by import By


driver.get("http://127.0.0.1:8000/pages/day4_advanced.html")

download_link = driver.find_element(By.ID, "download-template")

assert "day4_upload_template.txt" in download_link.get_attribute("href")
assert download_link.get_attribute("download") == ""
```

#### 这段代码的重点不是“验证文件已落盘”

而是：

- 我能找到下载链接
- 我知道它要下载什么文件
- 我知道下一阶段如果要验证落盘，需要再做浏览器配置

---

### 6.4 下载真要做自动化时通常还需要什么

真做下载自动化时，常见还要加：

1. 配浏览器默认下载目录
2. 关闭保存确认弹窗
3. 点击下载后去文件系统检查目标文件是否出现
4. 测试前后清理目录

这已经更像“项目实战”内容了，所以 Day 4 补充专题先不把你带得太深。

---

### 6.5 多标签页 / 多窗口先建立整体认知

在 Selenium 里：

- 标签页
- 新窗口

统一都按 `window` 处理。

所以这块最重要的是“窗口上下文切换”，而不是“标签页是不是一种特殊元素”。

最常用的 4 个 API 是：

- `driver.current_window_handle`
- `driver.window_handles`
- `driver.switch_to.window(...)`
- `driver.close()`

`driver.quit()` 也相关，但它是“结束整个浏览器会话”，不是“切窗口”。

---

### 6.6 `driver.current_window_handle`

#### 它是什么

这是一个**属性**，不是方法。

写法：

```python
origin = driver.current_window_handle
```

#### 它返回什么

返回当前标签页 / 当前窗口的句柄，类型通常可以理解成字符串。

你可以把它理解成：

`当前这个页面自己的身份证号。`

#### 什么时候用

最常见用法是：

1. 在打开新标签页之前先把当前页记下来
2. 后面关掉新页后再切回来

#### 最容易错在哪里

很多人会把它写成：

```python
driver.current_window_handle()
```

这是不对的，因为它不是方法，没有 `()`。

---

### 6.7 `driver.window_handles`

#### 它是什么

这也是一个**属性**，不是方法。

写法：

```python
all_handles = driver.window_handles
```

#### 它返回什么

返回当前浏览器会话里**全部标签页 / 窗口的句柄列表**。

#### 什么时候用

最常见用法是：

- 判断新页有没有真的打开
- 找出“不是原页面”的那个句柄

例如：

```python
wait.until(lambda d: len(d.window_handles) == 2)
```

#### 最容易错在哪里

很多人会直接写：

```python
driver.switch_to.window(driver.window_handles[-1])
```

学习阶段有时能跑，但它的问题是：

- 你默认假设“最后一个一定是目标页”
- 如果后面浏览器里页签多了，你会越来越乱

更稳的思路是：

```python
new_handle = [h for h in driver.window_handles if h != origin][0]
```

明确区分原句柄和新句柄。

---

### 6.8 `driver.switch_to.window(handle)`

#### 它是什么

这是一个**方法**。

写法：

```python
driver.switch_to.window(new_handle)
```

#### 参数是什么

最常见传入的是：

- 某个窗口句柄

#### 返回什么

这个方法本身通常不拿来接返回值。

你要把它理解成：

`执行一个“把当前上下文切到别的窗口”的动作。`

#### 什么时候用

当新页已经打开，但 Selenium 还停留在旧页面上下文时，就必须切。

不切的话，新页面即使肉眼已经看见了，你接下来写：

```python
driver.find_element(...)
```

仍然是在旧页面上下文里找。

#### 最容易报什么错

- `NoSuchWindowException`

常见原因：

- 你切到一个已经关闭的窗口
- 句柄写错
- 页签其实还没打开，就急着切

所以这块通常要搭配：

```python
wait.until(lambda d: len(d.window_handles) == 2)
```

先等数量变多，再切。

---

### 6.9 `driver.close()` 和 `driver.quit()`

这两个方法特别容易混。

#### `driver.close()`

作用：

`只关闭当前标签页 / 当前窗口`

最常见搭配：

```python
driver.close()
driver.switch_to.window(origin)
```

也就是：

1. 关掉当前新页
2. 再切回原页

#### `driver.quit()`

作用：

`关闭整个浏览器会话`

也就是说：

- 所有标签页一起关
- 浏览器驱动也结束

它更适合放在：

- 测试结束
- `fixture` 清理阶段

例如：

```python
@pytest.fixture
def driver():
    browser = webdriver.Edge()
    yield browser
    browser.quit()
```

#### 一句话区分

- `close()`：关当前页
- `quit()`：关整个浏览器

---

### 6.10 多标签页切换完整示例

```python
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait


driver.get("http://127.0.0.1:8000/pages/day4_advanced.html")

origin = driver.current_window_handle

driver.find_element(By.ID, "open-dashboard-tab").click()

wait = WebDriverWait(driver, 10)
wait.until(lambda d: len(d.window_handles) == 2)

new_handle = [handle for handle in driver.window_handles if handle != origin][0]
driver.switch_to.window(new_handle)

assert "day4_dashboard" in driver.current_url

driver.close()
driver.switch_to.window(origin)

assert "day4_advanced" in driver.current_url
```

#### 这段代码要读出什么

1. `origin` 是原页面句柄
2. 点击后不会自动切到新页
3. 要先等新句柄出现
4. 再手动切过去
5. 关闭新页后要再切回原页

## 8. 一页速查表

### 8.1 iframe

- `driver.switch_to.frame(...)`
  - 切进 iframe
- `driver.switch_to.parent_frame()`
  - 回上一级 frame
- `driver.switch_to.default_content()`
  - 回最外层主页面
- `EC.frame_to_be_available_and_switch_to_it(...)`
  - 等 frame 可用并切进去

### 8.2 原生弹窗

- `driver.switch_to.alert`
  - 获取弹窗对象
- `alert.text`
  - 读弹窗文本
- `alert.accept()`
  - 点确定
- `alert.dismiss()`
  - 点取消
- `alert.send_keys("xxx")`
  - 给 prompt 输入内容

### 8.3 上传下载

- `file_input.send_keys(绝对路径)`
  - 文件上传
- `download_link.get_attribute("href")`
  - 看下载地址
- `download_link.get_attribute("download")`
  - 看下载属性

### 8.4 窗口切换

- `driver.current_window_handle`
  - 当前窗口句柄
- `driver.window_handles`
  - 全部窗口句柄列表
- `driver.switch_to.window(handle)`
  - 切到目标窗口
- `driver.close()`
  - 关当前窗口
- `driver.quit()`
  - 关整个浏览器
