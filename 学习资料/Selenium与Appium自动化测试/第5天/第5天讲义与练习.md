## 4. 重点一：Appium 三层结构

### 4.1 一句话理解

Appium 客户端不直接控制手机，它要通过服务端和平台驱动。

### 4.2 三层分别是什么

- Python 客户端
- Appium 服务
- 平台驱动：Android `UiAutomator2` / iOS `XCUITest`

### 4.3 随堂检查

1. Appium Python 代码是不是直接操控设备？
2. Android 和 iOS 可不可以共用同一个平台驱动？

### 4.4 答案

1. 不是，它要经过 Appium 服务和平台驱动。
2. 不可以。

## 5. 重点二：会话和能力参数

### 5.1 一句话理解

能力参数就是告诉 Appium：

- 连哪个平台
- 用哪个驱动
- 打开哪个 App

### 5.2 Android 最小版本

```python
options = UiAutomator2Options()
options.platform_name = "Android"
options.automation_name = "UiAutomator2"
options.device_name = "Android Emulator"
options.app_package = "io.appium.android.apis"
options.app_activity = ".ApiDemos"
```

### 5.3 随堂检查

1. 哪个字段在指定平台？
2. 哪个字段在指定 Android 驱动？
3. 哪两个字段在指定 App 入口？

### 5.4 答案

1. `platform_name`
2. `automation_name`
3. `app_package` 和 `app_activity`

## 6. 重点三：第一个脚本

### 6.1 最小骨架

```python
from appium import webdriver
from appium.options.android import UiAutomator2Options
from appium.webdriver.common.appiumby import AppiumBy
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC
```

你要先记住：

- `webdriver` 来自 Appium，不是 Selenium
- 移动端定位优先用 `AppiumBy`
- 等待仍然直接复用 Selenium 的 `WebDriverWait`

### 6.2 核心流程

1. 准备 options
2. 建立 `webdriver.Remote(...)`
3. 等待元素出现
4. 点击元素
5. 断言下一屏结果

### 6.3 随堂检查

1. 为什么等待类还在用 Selenium 的？
2. `AppiumBy` 和 Web 里的 `By` 最大区别是什么？

### 6.4 答案

1. 因为 Appium 客户端沿用了 WebDriver 思路和很多 Selenium 能力。
2. `AppiumBy` 支持更多移动端专属定位策略。

## 7. 重点四：移动端定位优先级

### 7.1 推荐顺序

- `ACCESSIBILITY_ID`
- 稳定 `ID`
- 最后才 `XPATH`

### 7.2 为什么先 accessibility id

因为它通常：

- 语义更明确
- 跨平台更统一
- 比结构型 XPath 更稳

### 7.3 随堂检查

1. Day 5 最推荐先尝试哪种定位？
2. XPath 在移动端是不是默认最优先？

### 7.4 答案

1. `ACCESSIBILITY_ID`
2. 不是。
