# Day 5 运行说明

这份说明对应 Day 5 的教材和讲义。

今天不额外提供单独的 `.py` 示例代码文件，练习方式是：

- 使用你自己的 Android 模拟器或真机
- 跟着教材里的代码片段自己落地
- 先跑通 Android 主线，再对照理解 iOS 写法

## 1. 先检查 Appium 环境

```bash
node -v
npm -v
appium -v
```

如果未安装：

```bash
npm install -g appium
```

如果你用的是本地安装：

```bash
npm install appium
```

那后面的命令要写成：

```bash
npx appium
npx appium setup mobile
npx appium driver list --installed
```

## 2. 安装移动驱动

推荐：

```bash
appium setup mobile
```

或分别安装：

```bash
appium driver install uiautomator2
appium driver install xcuitest
```

安装后检查：

```bash
appium driver list --installed
```

## 3. 启动设备

Android 检查：

```bash
adb devices
```

iOS 模拟器检查：

```bash
xcrun simctl list devices
```

## 4. 启动 Appium 服务

```bash
appium
```

默认连接地址：

- `http://127.0.0.1:4723`

如果是本地安装路线，就用：

```bash
npx appium
```

## 4.1 Appium Inspector 怎么接

Day 5 做定位时，建议一定配合 `Appium Inspector`。

如果你还没装，它是一个单独的桌面工具，常见下载地址：

- `https://github.com/appium/appium-inspector/releases`

最小流程：

1. 先起好 Appium 服务
2. 确认设备在线
3. 打开 `Appium Inspector`
4. 填入服务地址：`http://127.0.0.1:4723`
5. 填入和代码一致的 capabilities
6. 成功连上后，在左侧元素树点选目标元素
7. 在右侧属性面板看：
   - Android：`content-desc`、`resource-id`、`text`
   - iOS：`name`、`label`、`value`

如果你想找 `ACCESSIBILITY_ID`，优先看：

- Android：`content-desc`
- iOS：`name` 或显式 accessibility identifier

## 5. Python 客户端依赖

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -U pip
pip install Appium-Python-Client pytest
```

## 6. Day 5 推荐最小验证链路

1. Appium 服务能起
2. `uiautomator2` 已安装
3. Android 设备在线
4. Python 代码能成功 `webdriver.Remote(...)`
5. 目标 App 能启动
6. 一个元素能被定位和点击

## 7. 今天如果跑不通，优先按这个顺序排查

1. Appium 服务是否真的在监听 `127.0.0.1:4723`
2. 目标驱动是否已安装
3. 设备是否在线
4. `app_package/app_activity` 或 `bundle_id` 是否写对
5. Python 客户端是否装对
6. 当前定位方式是否真的适合这个元素
7. 你是否已经用 Inspector 确认过这个元素属性

## 8. 明确一下今天的主线

Day 5 的目标不是把所有移动端高级技巧都学完，而是先跑通：

- 服务
- 驱动
- 设备
- 会话
- 基础定位
- 基础点击与断言

只要这 6 个点打通，Day 6 就能接着学手势、权限、Toast、WebView 和 `pytest` 项目化。
