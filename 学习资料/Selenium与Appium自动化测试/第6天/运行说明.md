# Day 6 运行说明

这份说明对应 Day 6 的教材和讲义。

今天的重点不是重新搭环境，而是在 Day 5 环境基础上继续做：

- 平台特有定位
- 滚动与手势思路
- 移动端特殊场景
- Appium + pytest
- 移动端 POM

## 1. 沿用 Day 5 环境

先确认下面这些还正常：

```bash
appium
```

另开终端确认：

```bash
adb devices
appium driver list --installed
```

## 2. Day 6 推荐最小验证链路

1. Appium 服务正常
2. Android 设备在线
3. App 能拉起
4. 用 `pytest` 跑一条 Appium 用例
5. 做一次滚动查找
6. 识别一个移动端特殊场景

## 3. Day 6 代码组织建议

如果你今天开始想做一点项目化，推荐结构：

```text
mobile_day6_project/
  conftest.py
  pages/
    home_page.py
  tests/
    test_home.py
```

核心思路还是：

- `conftest.py`：放 `mobile_driver`
- `pages/`：放页面对象
- `tests/`：放测试场景

## 4. 今天优先排查什么

如果 Day 6 场景跑不通，优先按这个顺序排查：

1. 当前是不是还在正确的 context
2. 元素是不是还没滚进可视区
3. 定位方式是不是平台不匹配
4. 权限弹窗是不是把业务页面挡住了
5. 你现在遇到的是页面问题，还是系统层问题

## 5. Day 6 学完后的建议

如果你这两天 Appium 内容走完，接下来最值得做的不是立刻堆更多高级 API，而是：

1. 选一个真实 App 场景
2. 写 3 到 5 条稳定用例
3. 用 `pytest + fixture + POM` 把它们组织起来
4. 再根据真实重复去抽公共方法

这样比继续堆“知识点收藏”更有价值。
