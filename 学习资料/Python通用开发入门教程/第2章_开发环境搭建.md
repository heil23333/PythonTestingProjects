# 第 2 章：开发环境搭建

## 1. 这一章为什么重要

很多初学者不是卡在“不会写代码”，而是卡在“代码还没学，环境先出问题了”。

比如：

- Python 安装了，但命令行找不到
- IDE 装好了，但运行按钮点了没反应
- 明明写了代码，却不知道文件保存在哪里

这些问题如果不先解决，后面学习会反复被打断。

所以这一章的目标很明确：

**让你把 Python 开发最基础的运行环境搭起来，并成功执行第一个程序。**

---

## 2. 你到底需要安装什么

一个最基础的 Python 学习环境，通常只需要下面几样东西：

### 2.1 Python 解释器

这是核心。

没有它，你写的 Python 代码就没法执行。

### 2.2 代码编辑器或 IDE

常见选择有：

- VS Code
- PyCharm

如果你是初学者，二者都可以。你可以这样理解：

- VS Code 更轻量，扩展能力强
- PyCharm 对 Python 更“开箱即用”

### 2.3 终端或命令行工具

你后面会频繁用到终端执行命令，例如：

```bash
python --version
python hello.py
pip --version
```

所以你不需要害怕命令行，它不是高级工具，而是 Python 学习中的基础工具。

---

## 3. 安装 Python 的基本步骤

不同操作系统界面会有差异，但总体步骤很接近。

### 3.1 下载 Python

建议从 Python 官方网站下载安装包，并选择 Python 3 的稳定版本。

下载时注意两件事：

- 不要下载过旧版本
- 不要混用来源不明的安装包

### 3.2 安装时的关键点

安装过程中最重要的一件事，是确保 Python 可以在命令行里被找到。

在一些系统中，这通常意味着：

- 让安装器把 Python 加入环境变量

如果你错过了这一步，也不是完全没救，后面仍然可以手动配置，但会更麻烦一些。

### 3.3 安装完成后立即验证

安装完不要急着关掉界面，先打开终端，输入：

```bash
python --version
```

有些系统也可能需要输入：

```bash
python3 --version
```

如果你能看到类似下面的输出，就说明 Python 已经安装成功：

```text
Python 3.12.x
```

---

## 4. 什么是环境变量

初学者经常被“环境变量”这个词吓到，其实它没有那么神秘。

你可以这样理解：

当你在终端输入 `python` 时，系统需要知道“去哪里找这个程序”。

环境变量就是系统提供的一套“查找路径规则”。

如果 Python 安装路径已经被加入系统的查找路径，那么你在任何目录下输入：

```bash
python --version
```

系统都能找到 Python。

如果没有加入，你就可能会看到“命令不存在”或“无法识别该命令”之类的报错。

---

## 5. 如何验证安装是否成功

至少做下面 3 个验证。

### 5.1 验证 Python 版本

```bash
python --version
```

### 5.2 验证 pip

`pip` 是 Python 常用的包管理工具，后面安装第三方库要用它。

```bash
pip --version
```

如果这条命令也能正常返回版本信息，说明你的环境基础已经比较完整。

### 5.3 验证交互式解释器

在终端输入：

```bash
python
```

如果进入了一个可以逐行输入 Python 代码的界面，就说明解释器能正常工作。

例如你可以试一下：

```python
print("hello")
```

看到输出后，再输入：

```python
exit()
```

或按快捷键退出。

---

## 6. 安装 VS Code 或 PyCharm

你不一定非得同时安装两个，只要先选一个顺手的即可。

### 6.1 VS Code 的特点

- 启动快
- 界面简洁
- 需要安装 Python 扩展
- 适合同时学习多种语言或工具的人

### 6.2 PyCharm 的特点

- 对 Python 支持非常完整
- 项目结构感更强
- 调试体验好
- 安装体积通常更大

### 6.3 初学者怎么选

如果你没有明显偏好，可以遵循这个简单原则：

- 想轻量一些，用 VS Code
- 想更像“专业 Python 工具”，用 PyCharm

不管你选哪个，关键都不是“工具更强”，而是你能稳定打开、写代码、运行代码。

## 7. 创建第一个 Python 文件

假设你已经安装好编辑器，现在来创建第一个文件。

文件名可以叫：

```text
hello.py
```

注意两点：

- 文件扩展名是 `.py`
- 尽量不要用中文空格或奇怪符号做文件名

在文件中输入：

```python
print("Hello, Python!")
print("这是我的第一个 Python 程序。")
```

保存文件。

## 8. 运行 Python 程序的三种常见方式

### 8.1 方式一：在终端中运行

这是最推荐初学者尽早掌握的方式。

进入脚本所在目录后，执行：

```bash
python hello.py
```

优点是：

- 更接近程序真实运行方式
- 便于理解路径、环境和命令
- 后面学 `pip`、虚拟环境、项目运行时都离不开

### 8.2 方式二：在 IDE 中点击运行

这也是很常见的方式。

优点是：

- 操作直观
- 对初学者更友好
- 调试方便

但要注意，不要只会点按钮，不知道背后本质上还是“解释器执行脚本”。

### 8.3 方式三：在交互式解释器中逐行输入

适合：

- 快速试验一小段语句
- 临时验证某个语法

不适合：

- 写完整程序
- 长期保存代码

所以它更像“练习纸”，不是“正式作业本”。

## 9. 目录和路径的基本概念

初学者经常写对了代码，却因为“路径不对”运行失败。

先建立两个概念：

### 9.1 当前目录

你在终端里所在的位置，就是当前目录。

当你执行：

```bash
python hello.py
```

系统会优先在当前目录寻找 `hello.py`。

### 9.2 文件路径

路径就是文件所在的位置描述。

例如：

- 相对路径：相对于当前目录
- 绝对路径：从系统根位置开始的完整路径

初学阶段你只要先理解：

- “我现在在哪个目录”
- “我要运行的文件在哪个目录”

这两个问题经常决定程序能不能跑起来。

## 10. 第一个稍完整的示例

创建一个文件 `intro.py`，输入下面的内容：

```python
print("欢迎来到 Python 世界")
name = input("请输入你的名字：")
print("你好，" + name)
print("今天开始练习写程序。")
```

这个示例包含了几个重要动作：

- 输出文字
- 接收用户输入
- 保存到变量
- 再把结果输出出来

如果你把这个文件跑通，就说明你已经完成了真正意义上的“第一次程序交互”。

## 11. 常见问题与排查思路

### 11.1 问题一：终端提示找不到 `python`

常见原因：

- Python 没安装成功
- 安装了，但没有加入环境变量
- 系统实际可用命令是 `python3`

排查思路：

1. 先试 `python --version`
2. 再试 `python3 --version`
3. 如果都不行，回头检查安装和环境变量

### 11.2 问题二：文件明明写了，运行时却说找不到文件

常见原因：

- 当前目录不对
- 文件名拼错
- 扩展名不是 `.py`

排查思路：

1. 确认你当前所在目录
2. 确认文件真实名称
3. 确认是不是写成了 `hello.py.txt`

### 11.3 问题三：点 IDE 运行没有反应

常见原因：

- 没有正确选择 Python 解释器
- 文件没有保存
- 编辑器插件或配置未完成

先不要急着怀疑代码，优先检查运行配置。

### 11.4 问题四：中文显示乱码

这通常和终端编码或文件编码有关。

初学阶段建议：

- 文件默认使用 UTF-8 编码
- 尽量使用主流编辑器默认设置

---

## 12. 常见错误提醒

### 12.1 只在 IDE 里点运行，不理解命令行

短期看似乎没问题，但后面很容易在环境、路径、依赖这些问题上反复卡住。

### 12.2 把安装环境和写代码混为一谈

环境问题和代码问题常常是两类不同问题。你要学会区分：

- 是代码写错了
- 还是 Python 根本没装好

### 12.3 一次装太多工具

初学阶段只要把 Python、一个编辑器、一个终端运行流程搞定就够了，不要一开始就装一堆插件和高级工具。
