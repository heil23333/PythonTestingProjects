- 而是为了后面失败时有结果载体可看

报告文件会生成在：


你后面每次跑完，至少要会看这 4 件事：

1. 总共跑了多少条测试
2. 哪些通过，哪些失败
3. 失败停在哪条测试
4. 失败停在哪一行

也就是说：

- 先学会生成

## 5. 讲义模块二：给测试打标记

### 5.1 改 `pytest.ini`

```ini
[pytest]
addopts = -v
testpaths = testcases
python_files = test_*.py
python_functions = test_*
markers =
    smoke: core flow smoke tests
    regression: broader regression tests
```

### 5.2 给测试加标记

```python
import pytest


@pytest.mark.smoke
def test_health():
    ...
```

```python
import pytest


@pytest.mark.regression
def test_product_create_query_delete(auth_headers):
    ...
```

### 5.3 运行标记测试

只跑冒烟：

```bash
pytest -m smoke
```

只跑回归：

```bash
pytest -m regression
```

如果你想只跑冒烟测试并顺手生成一份报告，也可以直接写：

```bash
pytest -m smoke --html=reports/smoke.html --self-contained-html
```

### 你要理解

- 标记的价值是让你“按目的跑测试”

### 5.4 标记放好以后到底怎么生效

真正生效不是因为你写了：

```python
@pytest.mark.smoke
```

就结束了。

而是因为你后面会配合：

```bash
pytest -m smoke
```

也就是说：

- 装饰器负责给测试分类
- `-m` 负责按分类筛选运行

## 6. 讲义模块三：给请求入口加日志

把 `common/request_handler.py` 改成下面这种思路：

```python
import requests

from common.logger import logger
from config.settings import TIMEOUT


def send_request(method, url, **kwargs):
    kwargs.setdefault("timeout", TIMEOUT)

    logger.info("request start | method=%s url=%s", method, url)
    logger.info("request kwargs | %s", kwargs)

    resp = requests.request(method=method, url=url, **kwargs)

    logger.info("response status | %s", resp.status_code)
    logger.info("response text | %s", resp.text)

    return resp
```

### 你要理解

- 最好不要每个测试自己 `print`
- 更好的方式是把日志放到公共请求层

### 6.1 这一层放好以后测试文件该怎么写

测试文件里不是直接写：

```python
logger.info(...)
```

而是继续正常调用：

```python
resp = send_request("GET", f"{BASE_URL}/health")
```

或者：

```python
resp = send_request(
    "POST",
    f"{BASE_URL}/api/v1/login",
    json={"username": "alice", "password": "123456"},
)
```

然后日志会自动在公共请求层里输出。

这一步的重点就是：

- 测试文件继续专注“我要测什么”
- 请求和响应日志由公共入口统一记录

### 随堂检查

1. 为什么请求日志更适合放在 `request_handler.py` 而不是每个测试文件？

### 参考答案

1. 因为这样可以统一记录请求过程，避免重复代码。

## 7. 讲义模块四：故意制造一个失败

随便找一条你已经通过的测试，例如健康检查：

原来：

```python
assert body["data"]["status"] == "ok"
```

改成：

```python
assert body["data"]["status"] == "bad"
```

再执行：

```bash
pytest -s -v --html=reports/report_fail.html --self-contained-html
```

### 你要观察什么

1. 终端里失败发生在哪一行
2. 日志里请求和响应是什么
3. 报告里这条测试是怎么显示的

这一步非常重要。
