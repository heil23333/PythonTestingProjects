# Python 接口自动化测试自学大纲

## 1. 学习目标

通过这份大纲，你的目标不是“会写几个接口用例”，而是逐步具备下面这些能力：

- 能独立阅读接口文档，并把文档转成可执行测试用例
- 能使用 Python 编写接口请求、断言、数据驱动和测试报告
- 能基于 `pytest` 搭建一个可维护的接口自动化测试框架
- 能处理登录鉴权、依赖接口、环境切换、测试数据管理等常见问题
- 能把接口自动化接入持续集成流程
- 能独立完成一个中小型项目的接口自动化测试实践

---

## 2. 学习路线总览

建议按下面顺序学习：

1. Python 基础
2. HTTP 和接口测试基础
3. Python 发起接口请求
4. `pytest` 测试框架
5. 接口自动化核心能力
6. 自动化框架设计
7. 持续集成与测试规范
8. 项目实战与简历沉淀

如果你是零基础，建议学习周期按 **12 周** 规划；如果你已经有 Python 基础，可以压缩到 **6 到 8 周**。

## 13. 建议的学习方法

### 13.1 正确顺序

不要一开始就学“框架搭建”，建议顺序是：

1. 先会手工测接口
2. 再会用 Python 调接口
3. 再学 `pytest`
4. 最后再搭框架

### 13.2 学习原则

- 每学一个知识点，必须马上写代码练习
- 不要只看视频不动手
- 不要上来就追求“高级框架”
- 先做能跑通的小项目，再追求优雅封装

### 13.3 推荐练习节奏

- 每天学习 1 到 2 小时
- 每周至少做一次总结
- 每两周整理一次代码结构
- 每月完成一个小项目
