## 4. 讲义模块一：先写一个最简单测试

```python
def test_demo():
    assert 1 + 1 == 2
```

### 你要理解

- `test_` 是测试函数命名入口
- `assert` 是测试的核心

### 随堂检查

1. 为什么 `print(1 + 1)` 不算测试？
2. 为什么 `assert 1 + 1 == 2` 算测试？

### 参考答案

1. 因为它没有判断预期是否成立。
2. 因为它明确表达了预期，并且失败时会报错。

## 5. 讲义模块二：健康检查测试

```python
import requests


def test_health():
    resp = requests.get("http://127.0.0.1:5001/health")
    body = resp.json()

    assert resp.status_code == 200
    assert body["code"] == 0
    assert body["data"]["status"] == "ok"
```

### 你要理解

- 一个接口测试至少应该有 2 到 3 个关键断言

### 随堂检查

1. 这里为什么不能只断言 `status_code == 200`？

### 参考答案

1. 因为 HTTP 成功不代表业务一定成功，还要检查返回内容。

## 6. 讲义模块三：登录参数化测试

```python
import pytest
import requests


@pytest.mark.parametrize(
    "username,password,expected_status,expected_code",
    [
        ("alice", "123456", 200, 0),
        ("alice", "wrong", 401, 10014),
        ("", "123456", 400, 10011),
    ],
)
def test_login(username, password, expected_status, expected_code):
    resp = requests.post(
        "http://127.0.0.1:5001/api/v1/login",
        json={"username": username, "password": password},
    )
    body = resp.json()

    assert resp.status_code == expected_status
    assert body["code"] == expected_code
```

### 你要理解

- 参数化的本质是“同一逻辑，多组数据”

### 随堂检查

1. 为什么这里适合参数化？
2. 如果把 10 组登录测试都写成 10 个函数，会有什么问题？

### 参考答案

1. 因为接口和断言结构相同，只是输入和预期不同。
2. 重复代码太多，维护成本高。

## 7. 讲义模块四：fixture 获取 token

```python
import pytest
import requests


@pytest.fixture
def login_token():
    resp = requests.post(
        "http://127.0.0.1:5001/api/v1/login",
        json={"username": "alice", "password": "123456"},
    )
    return resp.json()["data"]["token"]
```

### 你要理解

- fixture 是公共前置数据提供器

### 随堂检查

1. 为什么 token 很适合做 fixture？

### 参考答案

1. 因为很多测试都会依赖登录后的 token，写成 fixture 可以复用。

## 8. 讲义模块五：带 token 的资料测试

```python
import requests


def test_profile(login_token):
    headers = {"Authorization": f"Bearer {login_token}"}
    resp = requests.get("http://127.0.0.1:5001/api/v1/profile", headers=headers)
    body = resp.json()

    assert resp.status_code == 200
    assert body["code"] == 0
    assert body["data"]["username"] == "alice"
```

### 你要理解

- 这里 `login_token` 不是普通变量，而是由 fixture 注入进来的

### 随堂检查

1. 如果这里没有 fixture，会怎么写？
2. 为什么 fixture 能减少重复？

### 参考答案

1. 每个测试都要自己再写一遍登录代码。
2. 因为公共前置只写一次。

## 9. 讲义模块六：列表查询测试

```python
import requests


def test_product_list():
    resp = requests.get(
        "http://127.0.0.1:5001/api/v1/products",
        params={"page": 1, "page_size": 3},
    )
    body = resp.json()

    assert resp.status_code == 200
    assert body["code"] == 0
    assert body["data"]["page"] == 1
    assert len(body["data"]["items"]) <= 3
```

### 你要理解

- 列表接口不只是断言成功，还要断言结构和字段
