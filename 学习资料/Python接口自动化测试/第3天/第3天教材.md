## 4. pytest 是什么

`pytest` 是 Python 最常用的测试框架之一。

你现在先把它理解成：

- 它帮你执行测试代码
- 它帮你判断通过还是失败
- 它帮你组织测试结构

Day 3 的重点不是写“业务框架”，而是学会按测试框架的方式思考。

## 5. 最基础的测试函数

最简单的 `pytest` 测试长这样：

```python
def test_add():
    assert 1 + 1 == 2
```

这里你先记两件事：

- 测试函数名通常以 `test_` 开头
- `assert` 就是断言

---

## 6. 为什么接口脚本不等于测试

很多初学者在 Day 2 会写出这种代码：

```python
resp = requests.get("http://127.0.0.1:5001/health")
print(resp.status_code)
print(resp.json())
```

它是脚本，不是测试。

因为它没有明确判断“预期是什么”。

真正测试应该写成：

```python
import requests


def test_health():
    resp = requests.get("http://127.0.0.1:5001/health")
    resp_json = resp.json()

    assert resp.status_code == 200
    assert resp_json["code"] == 0
    assert resp_json["data"]["status"] == "ok"
```

这才叫测试。

## 7. assert 怎么用

Day 3 最重要的语法就是 `assert`。

### 7.1 基本写法

```python
assert 1 == 1
assert "success" == "success"
```

### 7.2 接口测试里的常见断言

```python
assert resp.status_code == 200
assert resp_json["code"] == 0
assert resp_json["message"] == "success"
assert "token" in resp_json["data"]
```

### 7.3 断言要有层次

建议至少分成两层：

1. HTTP 层断言
2. 业务层断言

例如登录接口：

```python
assert resp.status_code == 200
assert resp_json["code"] == 0
assert resp_json["data"]["user"]["username"] == "alice"
```

## 9. 参数化是什么

参数化是 `pytest` 里非常重要的能力。

它解决的问题是：

- 同一个接口
- 一样的测试逻辑
- 不同的输入数据

你不需要把代码复制很多份。

### 9.1 例子

```python
import pytest


@pytest.mark.parametrize(
    "username,password,expected_status,expected_code",
    [
        ("alice", "123456", 200, 0),
        ("alice", "wrong", 401, 10014),
        ("", "123456", 400, 10011),
    ],
)
def test_login(username, password, expected_status, expected_code):
    ...
```

### 9.2 你需要理解的点

- 这不是魔法
- 本质上就是“同一套逻辑跑多组数据”

## 10. fixture 是什么

fixture 是 `pytest` 的前置处理机制。

你可以先把它理解成：

- 给测试函数准备公共数据
- 避免重复写相同前置步骤

### 10.1 最常见场景

登录拿 token。

如果每个测试都自己写一遍登录，很重复。

这时就适合用 fixture：

```python
import pytest
import requests


@pytest.fixture
def login_token():
    resp = requests.post(
        "http://127.0.0.1:5001/api/v1/login",
        json={"username": "alice", "password": "123456"},
    )
    return resp.json()["data"]["token"]
```

然后测试里直接用：

```python
def test_profile(login_token):
    headers = {"Authorization": f"Bearer {login_token}"}
    ...
```

    [

    )
    body = resp.json()

    assert body["code"] == expected_code

    if expected_code == 0:
        assert body["message"] == "success"
        assert "token" in body["data"]

### 19.3 `test_profile.py` 完整示例

import requests

def test_profile(login_token):
    headers = {"Authorization": f"Bearer {login_token}"}
    resp = requests.get(f"{BASE_URL}/api/v1/profile", headers=headers)

    assert body["message"] == "success"

### 19.4 `test_product_list.py` 完整示例

```python
import requests


BASE_URL = "http://127.0.0.1:5001"


def test_product_list():
    resp = requests.get(
        f"{BASE_URL}/api/v1/products",
        params={"page": 1, "page_size": 3},
    )
    body = resp.json()

    assert resp.status_code == 200
    assert body["code"] == 0
    assert body["data"]["page"] == 1
    assert body["data"]["page_size"] == 3
    assert isinstance(body["data"]["items"], list)
    assert len(body["data"]["items"]) <= 3
```

### 20.3 如果故意写错断言

比如把：

```python
assert resp.status_code == 200
```

故意改成：

```python
assert resp.status_code == 201
```

那 `pytest` 就会明确告诉你：

- 哪一条断言失败
- 实际值是什么
- 预期值是什么

这就是测试框架比普通脚本更有价值的地方。
