# 本地接口服务使用说明

## 1. 项目位置

本地服务项目在这里：

`/Users/Apple/Documents/Codes/PythonTestingProjects/练习项目/本地接口练习服务`

---

## 2. 作用

这套服务是为了让你在 Day 2 到 Day 4 期间有一套稳定可练的本地接口。

它主要覆盖：

- GET 请求
- POST 请求
- DELETE 请求
- `params`
- `json`
- `data`
- 登录鉴权
- token 传递
- 新增和删除链路

---

## 3. 启动步骤

进入项目目录：

```bash
cd /Users/Apple/Documents/Codes/PythonTestingProjects/练习项目/本地接口练习服务
```

创建虚拟环境：

```bash
python3 -m venv .venv
```

激活虚拟环境：

```bash
source .venv/bin/activate
```

安装依赖：

```bash
pip install -r requirements.txt
```

启动服务：

```bash
python app/app.py
```

启动成功后，服务地址为：

```text
http://127.0.0.1:5001
```

---

## 4. 如何确认服务正常

访问健康检查接口：

```bash
curl http://127.0.0.1:5001/health
```

如果返回类似下面内容，说明服务正常：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "status": "ok"
  }
}
```

## 6. 接口列表

### `GET /health`

作用：

- 检查服务是否启动

### `POST /api/v1/debug/reset`

作用：

- 重置商品和登录状态
- 每次练习前都可以调用一次，保证数据回到初始状态

### `POST /api/v1/login`

作用：

- 登录并获取 token

请求体：

```json
{
  "username": "alice",
  "password": "123456"
}
```

### `GET /api/v1/profile`

作用：

- 获取当前登录用户资料

请求头：

```http
Authorization: Bearer <token>
```

### `GET /api/v1/products`

作用：

- 查询商品列表

查询参数：

- `page`
- `page_size`
- `keyword`

### `GET /api/v1/products/<id>`

作用：

- 查询商品详情

### `POST /api/v1/products`

作用：

- 新增商品

请求头：

```http
Authorization: Bearer <token>
```

请求体：

```json
{
  "name": "Desk Lamp",
  "category": "lighting",
  "price": 199
}
```

### `DELETE /api/v1/products/<id>`

作用：

- 删除商品

请求头：

```http
Authorization: Bearer <token>
```

### `POST /api/v1/echo-form`

作用：

- 练习 `data` 表单提交

## 7. 推荐练习顺序

建议按下面顺序练：

1. `GET /health`
2. `GET /api/v1/products`
3. `POST /api/v1/login`
4. `GET /api/v1/profile`
5. `POST /api/v1/echo-form`
6. `POST /api/v1/products`
7. `DELETE /api/v1/products/<id>`

---

## 8. 常用调试命令

登录：

```bash
curl -X POST http://127.0.0.1:5001/api/v1/login \
  -H "Content-Type: application/json" \
  -d '{"username":"alice","password":"123456"}'
```

查询商品列表：

```bash
curl "http://127.0.0.1:5001/api/v1/products?page=1&page_size=3"
```

表单提交：

```bash
curl -X POST http://127.0.0.1:5001/api/v1/echo-form \
  -d "name=alice" \
  -d "city=shanghai"
```

重置数据：

```bash
curl -X POST http://127.0.0.1:5001/api/v1/debug/reset
```

## 9. 使用建议

如果你今天只练 Day 2，建议不要上来就写大而全的代码。

正确顺序是：

1. 先用 `curl` 或 Postman 验证本地服务正常
2. 再用 Python `requests` 重写
3. 每个接口先单独写，再做链路串联

## 10. 常见问题

### 端口占用

如果 `5001` 端口被占用，你需要先结束旧服务，再重启。

### Flask 没装上

说明依赖还没安装好，回到虚拟环境里重新执行：

```bash
pip install -r requirements.txt
```

### 请求失败

优先检查：

- 服务是否启动
- 地址和端口是否正确
- 请求方法是否正确
- 是否传了必须的 token 或请求体
