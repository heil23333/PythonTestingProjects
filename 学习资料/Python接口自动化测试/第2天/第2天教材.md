## 4. requests 是什么

`requests` 是 Python 里最常用的 HTTP 请求库之一。

你可以先把它理解成：

- 它帮你发请求
- 它帮你拿响应
- 它帮你处理 `headers/params/json/data`

Day 2 的大部分代码，都会围绕它展开。

---

## 5. 第一次发送 GET 请求

今天最简单的入口，是先请求健康检查接口：

```python
import requests

resp = requests.get("http://127.0.0.1:5001/health")
print(resp.status_code)
print(resp.text)
print(resp.json())
```

你需要关注 3 个点：

- `status_code`：HTTP 状态码
- `text`：原始文本响应
- `json()`：把 JSON 响应转成 Python 字典

### 5.1 你应该看到什么

理论上你会看到类似结果：

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "status": "ok"
  }
}
```

这说明：

- HTTP 层成功
- 业务层也成功

## 6. params 是什么，什么时候用

`params` 用于 URL 查询参数。

今天你可以用商品列表接口练它：

```python
import requests

resp = requests.get(
    "http://127.0.0.1:5001/api/v1/products",
    params={"page": 1, "page_size": 3, "keyword": "mouse"},
)
print(resp.url)
print(resp.json())
```

### 6.1 你需要理解的点

- `params` 会被拼到 URL 后面
- 常用于查询接口
- 查询接口最容易练 `params`

### 6.2 常见错误

- 把查询参数误放到 `json`
- `page/page_size` 传字符串却以为是数字问题无所谓
- 不打印最终 URL，导致不知道请求到底发成了什么样

---

## 7. json 是什么，什么时候用

`json` 用于发送 JSON 格式请求体。

今天你可以用登录接口练它：

```python
import requests

resp = requests.post(
    "http://127.0.0.1:5001/api/v1/login",
    json={"username": "alice", "password": "123456"},
)
print(resp.status_code)
print(resp.json())
```

### 7.1 你需要理解的点

- `json` 会把 Python 字典按 JSON 方式发送
- 很多登录、新增、修改接口都会用它
- 如果服务要求 JSON，你却传 `data`，结果通常会不对

### 7.2 登录成功后拿什么

成功后你要从响应中提取：

- `data.token`

因为后面访问受保护接口时要用它。

---

## 8. data 是什么，什么时候用

`data` 常用于发送表单数据。

今天你可以用表单回显接口练：

```python
import requests

resp = requests.post(
    "http://127.0.0.1:5001/api/v1/echo-form",
    data={"name": "alice", "city": "shanghai"},
)
print(resp.json())
```

### 8.1 你需要先建立的理解

- `data` 常见于表单提交
- `json` 常见于 JSON 请求体
- `params` 常见于查询参数

Day 2 只要求你能区分使用场景，不要求你背底层编码细节。

## 9. 如何携带请求头

访问需要登录的接口时，通常要带 `Authorization` 请求头。

示例：

```python
import requests

login_resp = requests.post(
    "http://127.0.0.1:5001/api/v1/login",
    json={"username": "alice", "password": "123456"},
)
token = login_resp.json()["data"]["token"]

profile_resp = requests.get(
    "http://127.0.0.1:5001/api/v1/profile",
    headers={"Authorization": f"Bearer {token}"},
)
print(profile_resp.json())
```

### 9.1 这里你要学会什么

- 如何从响应中取字段
- 如何把字段放进后续请求头
- 如何完成最基础的接口关联

这就是后续自动化里非常常见的模式。

## 13. 常见错误与排查思路

### 13.1 连接失败

先检查：

- 本地服务是否启动
- 地址和端口是否写对
- 是否访问了 `127.0.0.1:5001`

### 13.2 登录失败

先检查：

- 是否传了 JSON
- 用户名密码是否正确
- 请求体字段名是否写对

### 13.3 访问资料接口返回 401

先检查：

- 是否先登录
- token 是否成功提取
- `Authorization` 格式是否为 `Bearer <token>`

### 13.4 新增商品失败

先检查：

- 是否先登录
- 是否传了 JSON
- `name/category/price` 是否都传了
