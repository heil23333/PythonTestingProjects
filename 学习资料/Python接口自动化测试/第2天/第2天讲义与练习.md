## 4. 讲义模块一：第一个 GET 请求

练习代码：

```python
import requests

resp = requests.get("http://127.0.0.1:5001/health")
print(resp.status_code)
print(resp.text)
print(resp.json())
```

你现在要看懂 3 件事：

- 请求地址写在哪里
- 响应状态码怎么拿
- JSON 响应怎么转成 Python 字典

### 随堂检查

1. `resp.status_code` 拿到的是什么？
2. `resp.json()` 返回的通常是什么类型？
3. 什么时候更适合打印 `resp.text`？

### 参考答案

1. HTTP 状态码
2. Python 字典或列表
3. 当你怀疑返回内容不是标准 JSON，或者想看原始响应时

## 5. 讲义模块二：练 params

练习代码：

```python
import requests

resp = requests.get(
    "http://127.0.0.1:5001/api/v1/products",
    params={"page": 1, "page_size": 2, "keyword": "accessory"},
)
print(resp.url)
print(resp.json())
```

### 你要理解

- `params` 会拼到 URL 上
- 列表查询接口很适合练 `params`
- 打印 `resp.url` 很重要

### 随堂检查

1. 为什么这个接口更适合用 `params` 而不是 `json`？
2. 如果 `page=abc`，你预期会发生什么？

### 参考答案

1. 因为这是查询接口，参数本质是查询条件。
2. 服务端应返回参数错误，通常是 `400`。

## 6. 讲义模块三：练 json

练习代码：

```python
import requests

resp = requests.post(
    "http://127.0.0.1:5001/api/v1/login",
    json={"username": "alice", "password": "123456"},
)
print(resp.json())
```

### 你要理解

- 登录接口通常更常用 `json`
- 响应中 token 是后续请求的关键数据

### 随堂检查

1. 为什么登录接口常用 `json`？
2. 登录成功后最关键的返回字段是什么？

### 参考答案

1. 因为登录属于提交数据场景，且现代接口常用 JSON 请求体。
2. `token`

## 7. 讲义模块四：练请求头和接口关联

练习代码：

```python
import requests

login_resp = requests.post(
    "http://127.0.0.1:5001/api/v1/login",
    json={"username": "alice", "password": "123456"},
)
token = login_resp.json()["data"]["token"]

profile_resp = requests.get(
    "http://127.0.0.1:5001/api/v1/profile",
    headers={"Authorization": f"Bearer {token}"},
)
print(profile_resp.json())
```

### 你要理解

- 前一个接口的响应字段，可以给后一个接口使用
- 这就是最基础的接口关联

### 随堂检查

1. 如果不带 `Authorization` 会怎样？
2. 如果 token 拼接格式错了会怎样？

### 参考答案

1. 一般会返回 `401`
2. 依然会鉴权失败

## 8. 讲义模块五：练 data

练习代码：

```python
import requests

resp = requests.post(
    "http://127.0.0.1:5001/api/v1/echo-form",
    data={"name": "alice", "city": "shanghai"},
)
print(resp.json())
```

### 你要理解

- `data` 更接近表单提交
- 它和 `json` 的使用场景不同

### 随堂检查

1. `data` 和 `json` 的最大区别是什么？
2. 为什么今天要单独练这个接口？

### 参考答案

1. 一个更偏表单数据，一个更偏 JSON 请求体。
2. 因为它能非常直观地帮助你区分两种传参方式。

## 9. 讲义模块六：练完整链路

练习目标：

1. 登录拿 token
2. 新增商品
3. 查询商品列表确认新增成功
4. 删除该商品

参考代码：

```python
import requests

base_url = "http://127.0.0.1:5001"

login_resp = requests.post(
    f"{base_url}/api/v1/login",
    json={"username": "alice", "password": "123456"},
)
token = login_resp.json()["data"]["token"]
headers = {"Authorization": f"Bearer {token}"}

create_resp = requests.post(
    f"{base_url}/api/v1/products",
    headers=headers,
    json={"name": "Desk Lamp", "category": "lighting", "price": 199},
)
product_id = create_resp.json()["data"]["id"]

list_resp = requests.get(f"{base_url}/api/v1/products", params={"page": 1, "page_size": 20})
print(list_resp.json())

delete_resp = requests.delete(
    f"{base_url}/api/v1/products/{product_id}",
    headers=headers,
)
print(delete_resp.json())
```

### 你要理解

- `requests` 不只是“发一个请求”，而是能串起一整条业务链路
