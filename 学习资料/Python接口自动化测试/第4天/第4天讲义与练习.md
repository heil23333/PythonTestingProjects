## 6. 讲义模块三：写 conftest.py

文件名：

```text
conftest.py
```

文件内容：

```python
import pytest
import requests

from config import BASE_URL, PASSWORD, TIMEOUT, USERNAME


@pytest.fixture
def reset_data():
    requests.post(f"{BASE_URL}/api/v1/debug/reset", timeout=TIMEOUT)


@pytest.fixture
def login_token(reset_data):
    resp = requests.post(
        f"{BASE_URL}/api/v1/login",
        json={"username": USERNAME, "password": PASSWORD},
        timeout=TIMEOUT,
    )
    body = resp.json()

    assert resp.status_code == 200
    assert body["code"] == 0

    return body["data"]["token"]


@pytest.fixture
def auth_headers(login_token):
    return {"Authorization": f"Bearer {login_token}"}
```

### 你要理解

- `reset_data` 负责让数据回到初始状态
- `login_token` 负责做登录关联
- `auth_headers` 负责把 token 整理成请求头

### 随堂检查

1. 为什么 `token` 不应该写死在配置文件里？

### 参考答案

1. 因为 token 是运行时动态产生的数据，不是固定配置。

## 7. 讲义模块四：写登录测试

文件名：

```text
test_login.py
```

文件内容：

```python
import json
from pathlib import Path

import pytest
import requests

from config import BASE_URL, TIMEOUT


DATA_FILE = Path(__file__).resolve().parent / "data" / "login_cases.json"
LOGIN_CASES = json.loads(DATA_FILE.read_text(encoding="utf-8"))


@pytest.mark.parametrize("case", LOGIN_CASES)
def test_login(case):
    resp = requests.post(
        f"{BASE_URL}/api/v1/login",
        json={
            "username": case["username"],
            "password": case["password"],
        },
        timeout=TIMEOUT,
    )
    body = resp.json()

    assert resp.status_code == case["expected_status"]
    assert body["code"] == case["expected_code"]
    assert body["message"] == case["expected_message"]
```

### 你要理解

- Day 3 是参数写在代码里
- Day 4 是参数写在数据文件里

## 8. 讲义模块五：写完整商品链路测试

文件名：

```text
test_product_flow.py
```

文件内容：

```python
import requests

from config import BASE_URL, TIMEOUT


def test_product_create_query_delete(auth_headers):
    create_resp = requests.post(
        f"{BASE_URL}/api/v1/products",
        headers=auth_headers,
        json={
            "name": "Desk Lamp",
            "category": "lighting",
            "price": 199,
        },
        timeout=TIMEOUT,
    )
    create_body = create_resp.json()

    assert create_resp.status_code == 201
    assert create_body["code"] == 0
    assert create_body["message"] == "success"

    product_id = create_body["data"]["id"]

    detail_resp = requests.get(
        f"{BASE_URL}/api/v1/products/{product_id}",
        timeout=TIMEOUT,
    )
    detail_body = detail_resp.json()

    assert detail_resp.status_code == 200
    assert detail_body["code"] == 0
    assert detail_body["data"]["id"] == product_id

    delete_resp = requests.delete(
        f"{BASE_URL}/api/v1/products/{product_id}",
        headers=auth_headers,
        timeout=TIMEOUT,
    )
    delete_body = delete_resp.json()

    assert delete_resp.status_code == 200
    assert delete_body["code"] == 0
    assert delete_body["data"]["deleted"] is True
```

### 你要理解

- 这就是 Day 4 最重要的接口关联练习
- 登录拿 token
- 新增拿 `id`
- 再去查询和删除

## 13. Day 4 最容易犯的错误

### 13.1 固定配置和动态数据混在一起

要记住：

- `BASE_URL` 是配置
- `token` 是动态数据

### 13.2 新增后不清理

这样数据会越来越乱。

### 13.3 断言太少

如果你只断一个状态码，Day 4 就没达到目标。

### 13.4 数据文件写出来但没真的用

这属于“形式上分离，实质上没用起来”。
