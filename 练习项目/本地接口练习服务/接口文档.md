# 本地接口练习服务接口文档

## 1. 文档说明

这份文档对应本地练习项目：

`/Users/Apple/Documents/Codes/PythonTestingProjects/练习项目/本地接口练习服务`

目标不是模拟完整业务系统，而是提供一套适合学习 Python 接口自动化测试的稳定接口。

默认服务地址：

```text
http://127.0.0.1:5001
```

统一响应结构：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

说明：

- HTTP 状态码表示协议层结果
- `code` 表示业务结果
- `message` 表示业务消息
- `data` 表示返回数据

## 3. 接口总览

| 接口 | 方法 | 说明 | 是否需要登录 |
| --- | --- | --- | --- |
| `/health` | `GET` | 健康检查 | 否 |

## 4. `GET /health`

### 作用

- 检查服务是否正常启动

### 请求示例

```bash
curl http://127.0.0.1:5001/health
```

### 成功响应

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "status": "ok"
  }
}
```

### 建议测试点

- 服务启动后请求成功
- 服务未启动时连接失败

---

## 5. `POST /api/v1/debug/reset`

### 作用

- 重置内存中的商品数据
- 清空当前登录 token
- 让练习环境回到初始状态

### 请求头

无强制要求。

### 请求体

无

### 请求示例

```bash
curl -X POST http://127.0.0.1:5001/api/v1/debug/reset
```

### 成功响应

```json
{
  "code": 0,
  "message": "data reset",
  "data": {
    "reset": true
  }
}
```

### 建议测试点

- 重置后返回成功
- 重置后新增数据应消失
- 重置后旧 token 应失效

## 6. `POST /api/v1/login`

### 作用

- 登录并返回 token

### 请求头

```http
Content-Type: application/json
```

### 请求体

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `username` | `string` | 是 | 用户名 |
| `password` | `string` | 是 | 密码 |

### 成功请求示例

```json
{
  "username": "alice",
  "password": "123456"
}
```

### 成功响应

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "token": "token-1",
    "user": {
      "id": 1,
      "username": "alice",
      "role": "student",
      "nickname": "Alice"
    }
  }
}
```

### 失败响应示例

用户名为空：

```json
{
  "code": 10011,
  "message": "username required",
  "data": null
}
```

密码为空：

```json
{
  "code": 10012,
  "message": "password required",
  "data": null
}
```

用户不存在：

```json
{
  "code": 10013,
  "message": "user not found",
  "data": null
}
```

密码错误：

```json
{
  "code": 10014,
  "message": "password error",
  "data": null
}
```

### HTTP 状态码

- 成功：`200`
- 参数错误：`400`
- 用户不存在：`404`
- 密码错误：`401`

### 建议测试点

- 正确账号密码登录成功
- 用户名为空
- 密码为空
- 用户不存在
- 密码错误
- 请求体不是 JSON

## 7. `GET /api/v1/profile`

### 作用

- 获取当前登录用户资料

### 请求头

```http
Authorization: Bearer <token>
```

### 请求示例

```bash
curl http://127.0.0.1:5001/api/v1/profile \
  -H "Authorization: Bearer token-1"
```

### 成功响应

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 1,
    "username": "alice",
    "role": "student",
    "nickname": "Alice"
  }
}
```

### 失败响应

未携带 token：

```json
{
  "code": 10021,
  "message": "token required",
  "data": null
}
```

### HTTP 状态码

- 成功：`200`
- 未认证：`401`

### 建议测试点

- 登录后访问成功
- 不带 token 访问
- 带错误 token 访问
- `Authorization` 格式错误

## 8. `GET /api/v1/products`

### 作用

- 查询商品列表

### 查询参数

| 参数 | 类型 | 必填 | 默认值 | 说明 |
| --- | --- | --- | --- | --- |
| `page` | `int` | 否 | `1` | 页码 |
| `page_size` | `int` | 否 | `5` | 每页条数 |
| `keyword` | `string` | 否 | 空字符串 | 名称或分类关键字 |

### 请求示例

```bash
curl "http://127.0.0.1:5001/api/v1/products?page=1&page_size=3&keyword=mouse"
```

### 成功响应

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "page": 1,
    "page_size": 3,
    "total": 1,
    "items": [
      {
        "id": 2,
        "name": "Wireless Mouse",
        "category": "mouse",
        "price": 129.0,
        "created_by": "system"
      }
    ]
  }
}
```

### 失败响应

页码不是整数：

```json
{
  "code": 10031,
  "message": "page and page_size must be int",
  "data": null
}
```

页码或分页条数小于等于 0：

```json
{
  "code": 10032,
  "message": "page and page_size must be positive",
  "data": null
}
```

### HTTP 状态码

- 成功：`200`
- 参数错误：`400`

### 建议测试点

- 默认分页查询
- 指定分页查询
- 关键字筛选
- `page` 非数字
- `page_size` 非正数
- 查询无数据

## 10. `POST /api/v1/products`

### 作用

- 新增商品

### 请求头

```http
Content-Type: application/json
Authorization: Bearer <token>
```

### 请求体

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `name` | `string` | 是 | 商品名称 |
| `category` | `string` | 是 | 商品分类 |
| `price` | `number` | 是 | 商品价格 |

### 请求示例

```json
{
  "name": "Desk Lamp",
  "category": "lighting",
  "price": 199
}
```

### 成功响应

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 7,
    "name": "Desk Lamp",
    "category": "lighting",
    "price": 199.0,
    "created_by": "alice"
  }
}
```

### 失败响应

未登录：

```json
{
  "code": 10051,
  "message": "token required",
  "data": null
}
```

请求体不是 JSON：

```json
{
  "code": 10052,
  "message": "json body required",
  "data": null
}
```

缺少名称：

```json
{
  "code": 10053,
  "message": "name required",
  "data": null
}
```

缺少分类：

```json
{
  "code": 10054,
  "message": "category required",
  "data": null
}
```

缺少价格：

```json
{
  "code": 10055,
  "message": "price required",
  "data": null
}
```

价格不是数字：

```json
{
  "code": 10056,
  "message": "price must be number",
  "data": null
}
```

### HTTP 状态码

- 成功：`201`
- 未认证：`401`
- 参数错误：`400`

### 建议测试点

- 登录后新增成功
- 未登录新增
- 缺少 `name`
- 缺少 `category`
- 缺少 `price`
- `price` 类型错误

## 12. `POST /api/v1/echo-form`

### 作用

- 练习 `data` 表单提交

### 请求体

表单格式，例如：

```text
name=alice&city=shanghai
```

### 请求示例

```bash
curl -X POST http://127.0.0.1:5001/api/v1/echo-form \
  -d "name=alice" \
  -d "city=shanghai"
```

### 成功响应

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "form": {
      "name": "alice",
      "city": "shanghai"
    }
  }
}
```

### 失败响应

```json
{
  "code": 10071,
  "message": "form data required",
  "data": null
}
```

### HTTP 状态码

- 成功：`200`
- 参数错误：`400`

### 建议测试点

- 正常表单提交
- 空表单提交
- 多字段表单提交
